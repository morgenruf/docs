Configuration

Configure Morgenruf by sending it direct messages. All settings are persisted per workspace.

DM Commands

Open a DM with @Morgenruf in Slack and send any of the following commands. Changes take effect immediately.

Set the standup channel

The channel where Morgenruf posts the daily standup summary.

config channel #standup
slack dm

Replace #standup with your actual channel name. Make sure you've also run /invite @Morgenruf in that channel so it can post there.

Set the standup time

The time at which Morgenruf DMs team members to collect their standup answers.

config time 09:00
slack dm

Uses 24-hour format. The time is interpreted in the workspace's configured timezone (see below).

Set the timezone

Standups are scheduled in this timezone. Use any IANA timezone name.

config timezone America/New_York
slack dm

Examples: Europe/Berlin, Asia/Tokyo, America/Los_Angeles, UTC

Set active days

Choose which days of the week Morgenruf should prompt for standups. Defaults to Monday through Friday.

config days mon,tue,wed,thu,fri
slack dm

Valid values: mon, tue, wed, thu, fri, sat, sun. Separate multiple days with commas.

Customize standup questions

By default Morgenruf asks three questions. You can replace them with anything your team prefers.

config questions
slack dm

Morgenruf will reply with an interactive prompt to enter up to three questions. The defaults are:

  1. What did you work on yesterday?
  2. What are you working on today?
  3. Any blockers or anything the team should know?

Example Setup Conversation

Here's what a typical first-time configuration looks like when DMing Morgenruf:

M
Morgenruf 9:02 AM
👋 Hey! Thanks for installing Morgenruf. I'm your async standup bot.

Let's get you set up. Send me config channel #your-channel to tell me where to post summaries.
Y
You 9:03 AM
config channel #engineering
M
Morgenruf 9:03 AM
✅ Standup summaries will be posted to #engineering.
Make sure to /invite @Morgenruf there if you haven't already.
Y
You 9:04 AM
config timezone Europe/Berlin
M
Morgenruf 9:04 AM
✅ Timezone set to Europe/Berlin.
Y
You 9:05 AM
config time 09:30
M
Morgenruf 9:05 AM
✅ Standup reminders will go out at 09:30 Europe/Berlin on weekdays.

You're all set! Want to run a test standup now? Just say @Morgenruf standup in any channel I'm in.

Environment Variables

If you're self-hosting, these environment variables control Morgenruf's behavior. Set them in your .env file or your container's environment.

Variable Required Description
SLACK_CLIENT_ID Required Your Slack app's Client ID. Found at api.slack.com/apps.
SLACK_CLIENT_SECRET Required Your Slack app's Client Secret. Keep this secret — do not commit to source control.
SLACK_SIGNING_SECRET Required Used to verify that incoming requests come from Slack. Found in app settings under Basic Information.
DATABASE_URL Required PostgreSQL connection string, e.g. postgresql://user:pass@host:5432/morgenruf
APP_URL Required Publicly accessible URL for this instance, e.g. https://api.morgenruf.dev. Used to construct OAuth redirect URIs.
RESEND_API_KEY Optional API key for Resend to send welcome emails. If omitted, emails are skipped.
PORT Optional HTTP port to listen on. Defaults to 3000.
⚠️

Keep your secrets safe. Never commit SLACK_CLIENT_SECRET or SLACK_SIGNING_SECRET to source control. Use a .env file (gitignored) or a secrets manager.

Help Command

At any time you can send Morgenruf a DM with just help to see a list of all available commands:

help
slack dm
